CodePipeline ECS deploy dev + production
========

This repository is an example implementation of CodePipeline to deploy ECS tasks
to separate services: development and production.

Development changes are deployed automatically on every push while changes to
production need to be accepted by someone with permissions.

Read more about this example here:
- [CodePipeline, CodeBuild, and multiple environments in ECS](https://pabis.eu/blog/2025-10-13-CodePipeline-CodeBuild-Multi-Environments-ECS.html)
- [On dev.to](https://dev.to/aws-builders/codepipeline-codebuild-and-multiple-environments-in-ecs-27l6)
